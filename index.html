Fluxo de Processsos - Parte 2

São criados os processos A, B e C, nesta ordem.

Para cada evento descrito, mostre quais são os processos que estarão prontos, executando ou bloqueados.

Lembre-se que:

  • Apenas um processo pode estar no estado “Executando”.
  • Se um processo liberar a CPU, outro deve ocupá-la imediatamente.
  • Assuma a ordem de chegada para definir quem ocupa a CPU.

Fluxo dos processos:

  1. Os processos A, B, C e D são criados.
  2. A CPU está livre e o processo A é despachado para execução.
  3. Processo A solicita operação de I/O e é bloqueado.
  4. Processo B é despachado para execução.
  5. Processo B solicita entrada via digitação e é bloqueado.
  6. Processo C é despachado para execução.
  7. Processo C conclui sua execução e termina.
  8. I/O do processo A é concluído; A volta para a fila de prontos.
  9. Digitação do processo B é concluída; B volta para a fila de prontos.
  10. Processo D é despachado para execução.
  11. Processo D solicita I/O e é bloqueado.
  12. Processo A é despachado novamente.
  13. Processo A conclui sua execução e termina.
  14. Processo B é despachado novamente.
  15. Processo B solicita nova operação de I/O e é bloqueado.
  16. I/O do processo D é concluído; D volta para a fila de prontos.
  17. Processo D é despachado e conclui sua execução.
  18. I/O do processo B é concluído; B volta para a fila de prontos.
  19. Processo B é despachado pela última vez.
  20. Processo B conclui sua execução e termina.

Monte uma tabela com colunas (Evento Nr., PRONTOS, EXECUTANDO, BLOQUEADOS) contendo os estados possíveis e para cada linha descreva um evento.

Descrição passo a passo do fluxo acima

  1. Todos os processos foram criados e foram alocados em prontos.
  2. A CPU está livre e devido a isso o processo A foi despachado para Executando.
  3. O processo A solicitou a operação de I/O e de devido a isso foi encaminhado para Bloqueado, até que a operação de I/O seja concluída.
  4. O processo B foi despachado para CPU pois ela sempre deve estar ocupada.
  5. O processo B solicitou a via de digitação, isso não pode ser realizado em executando e por isso foi encaminhado para Bloqueados.
  6. O processo C foi despachado para Executando, pois a CPU estava livre.
  7. O processo C concluiu sua execução e foi finalizado.
  8. O I/o do processo A foi concluído e ele está pronto para ser executado novamente, devido a isso ele vai para Prontos.
  9. A digitação do processo B foi concluída e ele está pronto para ser executado novamente, encaminhado para Prontos.
  10. O processo D é despachado para Executando pois a CPU estava vazia.
  11. O processo de solicita I/O e vai para Bloqueados para isso.
  12. O processo A é despachado novamente para Executando, pois a CPU estava vazia.
  13. O processo A termina sua execução e é finalizada.
  14. O processo B vai para Executando novamente.
  15. O processo B solicita uma nova I/O e é bloqueado novamente.
  16. O processo D concluído e volta para a fila de prontos.
  17. O processo D é despachado e conclui sua execução, o processo de foi finalizado.
  18. I/O do processo B é concluído e volta para a fila de Prontos.
  19. O processo B segue o fluxo e é despachado para Executando.
  20. O processo B finaliza sua execução e termina.
